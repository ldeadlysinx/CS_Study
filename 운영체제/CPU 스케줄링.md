# CPU 스케줄링
정의) 운영체제가 프로세스들에게 공정하고 합리적으로 CPU 자원을 배분하는것

공정한 CPU 스케줄링?
-~~사용하고 싶어하는 프로세스들에게 차례대로 할경우?~~
빨리 처리해야 하는 프로세스가 있기 때문에 좋은 방법은 아니다.
(프로세스마다 우선순위가 다르기 때문에)

**우선순위**
EX)입출력 작업이 많은 프로세스의 > CPU 작업이 많은 프로세스 

이유)입출력 프로세스는 CPU 프로세스에 비해 대기상태에 머물러있는 시간이 많으므로 CPU 사용시간이 적다 그러므로 우선적으로 처리한다음 CPU 집중 프로세스에 더 집중할수 있다.

-프로세스 우선순위는 PCB에 저장된다.

1. 스케줄링큐
	개요)운영체제가 PCB 를 일일히 확인해서 우선순위를 결정하는것은 비효유율 그러므로 스케줄링 큐 (FIFO)사용

	정의)특정 자원을 이용하고 싶은 프로세스들을 큐에 삽입해서 자원을 이용함

 - 준비 큐: CPU를 이용하기 위해 기다리는 중
 - 대기 큐:입출력장치를 이용하기 위해 기다리는 줄

2. 선점형 스케줄링
	 정의) 프로세스가 CPU를 비롯한 자원을 사용하고 있더라고 운영체제가 프로세스로부터 자원을 강제로 빼앗아 다른 프로세스에 할당할 수 있는 스케줄링 방식을 의미한다.

	장점) 어느 한 프로세스의 자원 독점을 막고 프로세스들에 골고루 자원을 배부할 수있다.
	
	단점) 문맥 교환 과정에서(문맥교환이 많을경우) 오버헤드가 발생
	
3. 비선점형 스케줄링
	정의) 하나의 프로세스가 자원을 사용하고 있다면 그 프로세스가 종료되거나 스스로 대기 상태에 접어들기 전까진 다른 프로세스가 끼어들 수 없는 스케줄링
	
	장점) 오베헤드가 선점형에서 적게 나타남

	단점) 당장 자원을 사용해야할 프로세스가 무작정 기다려야한다.

**스케줄링 알고리즘**

1. 선입 선처리 스케줄링(FCFS) 

	정의)단순히 준비 큐에 삽입된 순서대로 처리하는 비선점 스케줄링

	단점) 프레세스들이 기다리는 시간이 매우 길어질 수 있다(호위효과)

2. 최단 작업 우선 스케줄링
	
	정의) CPU사용 시간이 가장 짧은 프로세스부터 처리하는 스케줄 방식
	
3. 라운드 로빈 스케줄링
	정의) 선입 선처리 + 타임슬라이스(각 프로세스가 CPU를 사용할 수 있는 정해진  시간)

	타임 슬라이스가 너무 클경우 : 선입 선처리랑 다를게 없다.
	타임 슬라이스가 너무 작을경우:문맥 교환이 자주 일어나 오버헤드 발생

4. 최소 잔여 시간 우선 스케줄링
	정의)  최단 작업 우선 스케줄링 + 타임슬라이스

5. 우선순위 스케줄링
	정의)프로세스들에 우선순위를 부여하고, 우선순위가 높은 순서대로 CPU를 할당
	
	단점) 기아 현상 우선순위가 낮은 프로세스는 실행이 계속 연기됨

	해결책) 기아 현상을 방지하기 위해 에이징 기법 (대기 중인 프로세스의 우선순위를 마치 나이 먹듯 점차 증가시키는 방법)사용
6. 다단계 큐 스케줄링
	정의) 우선순위별로 준비 큐를 여러 개 사용하는 스케줄링 방식 우선순위가 가장 높은 큐가 비어 있으면 그 다음 우선순위 큐에 있는 프로세스 처리
	
	단점) 프로세스가 큐간의 이동 불가 기아 현상이 발생할 수 있다.
	
7. 다단계 피드백 큐 스케줄링
	정의) 큐 간의 이동이 가능하게 한다. 다단계  큐 스케줄링의 단점을 해결
	
	실행되고 있는 프로세스가 준비 상태에 갈경우 다음 우선순위 큐로 삽입된다.

	낮은 우선순위 큐에서 너무 오래 기다리고 있는 프로세스가 있다면 우선순위가 높은 큐로 이동시키는 에이징 기법을 사용한다. 기아현상 해결
